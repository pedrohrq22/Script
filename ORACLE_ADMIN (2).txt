ORACLE - ADMINISTRAÇÃO (sys/oracle_4U)

Sistema gerenciador de banco de dados

INSTACE [SGA - PROCESSOS] -> MEMÓRIA
BD -> DISCO (Conjunto de arquivos)

CADA BANCO DE DADOS TEM A SUA PRÓPRIA INSTÂNCIA.
UM BANCO PODE TER MAIS DE UMA INSTÂNCIA (MAIS DE UMA MÁQUINA).

COMPONENTES DA SGA:
Database buffer cache [UNDO] -> gravação das alterações para possível ROLLBACK.
Redo Log Buffer -> grava em disco as alterações, após commit ou até 3 segundos depois ou 1/3 do REDO cheio.
Shared Pool -> comandos executados recentemente.
Large Pool -> muito utilizada para realização de backup/restore.
Java Pool -> procedures em java.
Streams Pool -> para utilização de replicação.

PGA [área de memória não compartilhada]

--------------
DATABASE WRITER PROCESS (DBWn) -> processo de gravação de dados BUFFER -> REDO -> REDO LOG FILE | BUFFER -> UNDO | BD

REDO LOG FILE -> vários arquivos, há um processo chamado REDO LOG ARCHIVED, quando um log fica cheio, faz-se um backup, 
isto é, a cada troca de REDO LOG FILE é realizado um COMMIT no BD. 
Este processo de switch deve levar cerca de 20 a 30 minutos.

pode acontecer de termos uma possível recuperação através dos RED LOG ARQUIVADOS.

REDO LOG ARQUIVADO -> deve ser guardado por questão de auditoria, histórico ou possível retorno do banco para o passado.

----------------

Log Write Process (LGn) -> gravação de REDO NO DISCO

CHECK POINT -> ?

System Monitor Process (SMON)
faz recuperação do banco automático

Process Monitor Process (PMON) 
monitormento de sessões

Recoverer Process
Operacões com mais de um banco

Archiver Processes (ARCn) - gravação do REDO LOG ARCHIVE

DATABASE STORAGE ARCHITECTURE

Parameter File -> formato texto e binário(aconselhável)
Control Files -> controla a existência dos outros arquivos, não pode ser perdido.
Online redo log files
Data files
Archived redo log files
Backup files
Password file -> utilizado em conexões remotas e quando se loga com o sys.
Alert log and trace files -> histórico das alterações realizadas no banco.

-----------------------
LOGICAL AND PHYSICAL DATABASE STRUCTURES

Database -> Tablespace -> Segment -> Extent -> Oracle data block

		|
		V
		Datafile

Uma Tablespace deve apontar para um ou mais Datafile, cada Datafile pode ter até 32Gb.
Pode-se ter até 65 mil Datafiles em um Banco de Dados.

Na criação do banco de dados ele cria duas Tablespaces: SYSTEM e SYSAUX
O banco não funciona sem elas.

OBS.: Pode ter um Datafile de 2Tb, mas será possível criar apenas 1 Datafile por Tablespace.
